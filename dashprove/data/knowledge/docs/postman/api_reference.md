# Postman Sandbox API reference

Postman provides JavaScript APIs with the `pm` object, enabling you to test and access request and
response data in your test scripts run in the [Postman Sandbox][807]. You can use the `pm` object to
access request and response details, write assertions, and access and use variables. You can also
use the `pm` object to send HTTP requests and other meta information in the Postman Sandbox.

## [Use scripts to access cookies][808]

Use the `pm.cookies` methods in scripts to access and manipulate cookies. To learn more, see [Access
cookies in Postman scripts][809].

## [Use scripts with requests and responses][810]

Reference requests and responses with the `pm.request` and `pm.response` objects in scripts.
Streaming protocols also return a `pm.message` object. The `pm.info` object contains meta info
related to the request and script Use the `pm.sendRequest` method in your scripts to send requests
in Postman. To learn more, see [Reference Postman requests in scripts][811], [Use scripts to send
requests in Postman][812], [Reference request metadata in scripts][813] and [Reference message data
in scripts][814].

## [Use scripts with collections][815]

The `pm.execution` object provides information and context about requests and their responses during
a [collection run][816], such as sending requests or which request is running, its position in a
collection, and run-related metadata. To learn more, see [Use scripts in collection runs][817].

## [Script variables][818]

Access and manipulate different [variable types][819] and scopes in your scripts. To learn more, see
[Reference variables in Postman scripts][820].

## [Use scripts to visualize data][821]

The `pm.visualizer` object enables you to visually represent your API’s request responses with the
[Postman Visualizer][822]. To learn more, see [Script Postman visualizations][823].

## [Manage Postman Vault with scripts][824]

Access and manipulate [vault secrets][825] in your scripts with the `pm.vault` methods. To learn
more, see [Reference vault secrets in Postman scripts][826].

## [Test assertions with scripts][827]

Use the `pm.test` and `pm.expect` methods in your scripts to add test specifications and assertions.
To learn more, see [Writing tests and assertions in scripts][828].

## [Import packages into your scripts][829]

The `pm.require` method enables you to import packages from your team’s [Package Library][830] or
[external package registries][831] inside scripts in HTTP, gRPC, and GraphQL requests. To learn
more, see [Import packages into your scripts][832].
